# Argus Lab Training

---

## Purpose

This directory will contain training design, mentor-mode behavior, user progression, and future proficiency tracking plans for Argus Lab.

Argus Lab is designed to teach through experience, not memorization.

---

## Training Philosophy

Users should learn by working through realistic Linux failures.

The goal is not to hand users answers.

The goal is to help users develop troubleshooting judgment.

---

## Guidance Modes

Argus Lab may support multiple guidance modes.

### Independent Mode

The user works without assistance unless they ask for help.

### Assisted Mode

The system provides hints, investigation direction, and explanations without giving away the answer.

### Guided Resolution

The system walks through the issue when the user is stuck, explaining what went wrong and how to verify recovery.

---

## Mentor-Mode Direction

Argus Lab mentor mode should behave like a senior admin coaching a junior admin.

It should ask questions such as:

- What does the service state show?
- Which log would help confirm the issue?
- What command would narrow this down?
- What evidence supports your conclusion?
- How would you verify the fix?

The model should use grounded Argus diagnostic data to guide the learner progressively.

---

## Future Progression Tracking

Long term, Argus Lab may track:

- time to completion
- hints requested
- amount of guidance needed
- commands used
- accuracy of diagnosis
- quality of repair
- verification steps
- repeated mistakes
- improvement across similar scenarios

---

## Long-Term Direction

Future proficiency levels may include:

- junior Linux troubleshooter
- system administrator
- DevOps operator
- senior systems troubleshooter
- specialized infrastructure roles

The long-term goal is demonstrated ability, not memorization.
